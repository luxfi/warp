# Lux Warp

**Version 1.18.0** - Cross-chain messaging protocol with BLS signature aggregation

Warp is Lux Network's native cross-chain messaging (XCM) protocol, enabling secure and efficient communication between blockchains. It uses BLS signatures for efficient multi-signature aggregation and supports both EVM and non-EVM chains.

## Key Features

- **BLS Signature Aggregation**: Efficient multi-signature aggregation reduces on-chain verification costs
- **Post-Quantum Ready**: Designed with future post-quantum cryptography in mind
- **Chain-Agnostic**: Works with EVM chains, non-EVM chains, and custom VMs
- **Modular Design**: Components can be used independently
- **Protocol-First**: Clean separation between protocol definitions and implementations

## Architecture Overview

```
github.com/luxfi/warp
├── message.go         # Core message types (UnsignedMessage, Message)
├── signature.go       # BLS signature handling (BitSetSignature)
├── validator.go       # Validator set management
├── verifier.go        # Message verification interface
├── payload/           # Payload types (AddressedCall, Hash, etc.)
├── backend/           # Backend implementations
├── signer/            # Signing infrastructure
└── handler.go         # P2P message handling
```

## Quick Example

```go
import "github.com/luxfi/warp"

// Create an unsigned message
unsigned, err := warp.NewUnsignedMessage(
    networkID,
    sourceChainID,
    payload,
)

// Sign with validators
signed, err := warp.SignMessage(unsigned, signers, validators)

// Verify the message
err = warp.VerifyMessage(signed, networkID, validatorState, quorumNum, quorumDen)
```

## LP-118 Migration (v1.18.0)

As of v1.18.0, all functionality from `github.com/luxfi/p2p/lp118` has been consolidated into the warp package:

```go
// Old imports (deprecated)
import "github.com/luxfi/p2p/lp118"

// New imports
import "github.com/luxfi/warp"
```

See the [Signature Request API](/docs/api/signature-request) for complete migration guide.
